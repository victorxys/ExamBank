# 考试管理系统需求文档

## 1. 功能概述

本需求文档旨在描述一个考试管理系统，该系统将允许管理员创建、编辑、查看和管理考试，以及查看考试结果。系统将区分“考试”和“问卷/调研”两种类型的表单，确保考试流程的独立性和专业性。

## 2. 目标

*   提供一个直观的界面，用于创建和配置考试。
*   允许管理员添加、编辑和删除考试题目。
*   支持查看和管理考试结果。
*   确保考试数据与现有动态表单系统的数据分类保持一致。

## 3. 用户角色

*   **管理员 (Admin):**
    *   创建、编辑、删除考试。
    *   管理考试题目。
    *   查看所有考试的考试结果。
    *   配置考试属性（例如：考试时长、及格分数）。

## 4. 功能需求

### 4.1. 考试列表页面

*   显示所有已创建考试的列表。
*   每个考试应显示名称、描述、创建日期、状态（例如：草稿、已发布）、操作（编辑、删除、查看结果）。
*   支持按名称搜索考试。
*   支持分页显示。

### 4.2. 考试创建/编辑页面

*   **基本信息:**
    *   考试名称 (必填)
    *   考试描述 (可选)
    *   考试时长 (分钟，可选，默认为无限制)
    *   及格分数 (百分比，可选，默认为无及格分数)
    *   状态 (草稿/已发布)
*   **题目管理:**
    *   允许管理员从题库中选择现有题目添加到考试中。
    *   允许管理员在考试中直接创建新题目。
    *   支持对考试中的题目进行排序、编辑和删除。
    *   每个题目应包含：题目内容、题目类型（单选、多选、填空等）、选项（如果适用）、正确答案。
*   **保存/取消功能。**

### 4.3. 考试结果查看页面

*   显示特定考试的所有参与者的考试结果。
*   每个参与者应显示：姓名、提交时间、得分、是否及格、操作（查看详细结果）。
*   支持按参与者姓名搜索。
*   支持分页显示。
*   **详细结果查看:**
    *   显示参与者对每个题目的回答。
    *   显示每个题目的正确答案。
    *   标记回答正确或错误。

## 5. 非功能性需求

*   **性能:** 页面加载和数据操作应快速响应。
*   **安全性:** 确保只有授权用户才能访问和管理考试。
*   **可用性:** 界面应直观易用。
*   **兼容性:** 兼容主流浏览器。
*   **数据一致性:** 确保考试数据与动态表单数据保持一致。

## 6. 技术栈 (初步设想)

*   **后端:** Python, Flask, SQLAlchemy
*   **前端:** React, Material-UI (或现有项目使用的UI库)
*   **数据库:** PostgreSQL

## 7. 现有系统集成

*   利用现有动态表单系统的数据分类 (`form_type`) 来区分考试表单。
*   可能需要修改现有 `DynamicForm` 模型或创建新的 `Exam` 模型来存储考试特有的属性。
*   考试题目可以复用现有 `SurveyJS` 结构，但需要额外的字段来存储正确答案和评分逻辑。

## 8. 待定项

*   考试题目如何与 `SurveyJS` 结构结合，并存储正确答案。
*   考试评分逻辑的实现细节。
*   用户如何“参加”考试的流程设计。
*   是否需要独立的“题库管理”功能，还是直接在考试创建/编辑页面管理题目。
*   考试结果的统计分析功能。
