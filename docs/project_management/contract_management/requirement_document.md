# 合同管理微服务需求文档

## 1. 背景描述
为了实现“萌姨萌嫂客户管理系统”的模块化重构，需要建立独立的合同管理微服务。该服务负责管理所有客户与家政人员之间的合规性合同、试工合同等。

## 2. 功能目标
- **合约关联**：根据客户 ID 查询其名下的所有合同。
- **状态追踪**：支持查看合同的签署状态（待签署、已签署、已作废）和执行状态（正常、已终止、已续约）。
- **基本信息展示**：展示合同编号、合同类型、服务期限、合同金额等核心数据。
- **与客户详情集成**：在客户详情页面直观展示该客户的合同样本列表。

## 3. 关键约束
- **数据隔离**：合同数据存储在独立的 `contract_management_db` 中。
- **API 驱动**：前端通过调用微服务 API 获取数据，不直接查询数据库。
- **鉴权**：遵循现有的 JWT 鉴权机制。

## 4. 数据结构需求 (核心)
- **ID**: UUID (主键)
- **CustomerID**: UUID (关联客户)
- **ContractNumber**: 唯一编号
- **Type**: 合同类型 (育儿嫂正式、育儿嫂试工、月嫂正式等)
- **Status**: 状态 (草稿、有效、已失效)
- **StartDate / EndDate**: 服务周期
- **TotalAmount**: 合同总额
- **CreatedAt / UpdatedAt**: 时间戳
